## 디자인 레퍼런스 (WebFetch로 분석해봐)

* 레퍼런스 사이트 접근이 안되면 /reference_img 폴더 안의 사진을 참고해.
* 사이트들 랜딩 페이지 캡쳐사진이야. 

레퍼런스에서 가져올 방향성:
- 사진이 주인공. UI 요소는 최소화하고 여백을 넉넉하게
- 깔끔하고 고급스러운 무드. 화려한 컬러 금지
- 배경은 화이트 또는 아주 옅은 웜톤, 텍스트는 차분한 그레이/블랙
- 세리프 계열 로고 타이포 + 본문은 Pretendard
- 풀스크린 히어로 이미지 또는 그리드형 갤러리 중심 레이아웃

## 기술 스택
- Next.js (App Router) + Tailwind CSS
- Vercel 배포 전제
- next/image로 이미지 최적화 (포트폴리오 사이트라 성능이 중요)
- 실제 사진은 아직 없으니 placeholder 이미지로 레이아웃 먼저 구축.
  나중에 실제 사진으로 교체하기 쉬운 구조로 만들어줘 (예: 갤러리
  데이터를 별도 파일/폴더로 분리)

## 페이지 구성
1. Home: 풀스크린 히어로 + 대표 작업 미리보기
2. Portfolio: 갤러리 (카테고리: 본식스냅, 야외스냅 등. 블로그에서
   확인되는 카테고리 기준으로, 확인 안 되면 나한테 물어봐)
3. About: 업체 소개, 촬영 철학
4. Pricing: 상품/가격 안내 (내용은 placeholder)
5. Contact/Reservation: 문의 안내 (카카오톡 채널, 인스타그램 링크 자리)

## 요구사항
- 모바일 퍼스트. 예비 신부들이 대부분 아이폰으로 볼 거야
- 갤러리는 라이트박스(클릭 시 확대) 지원
- 한국어 사이트
- SEO 기본 세팅 (메타태그, OG 이미지 자리)
